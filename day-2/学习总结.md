# CUDA 矩阵乘法学习总结

## 1. 学习内容概述

本次学习主要围绕 CUDA 平台上的矩阵乘法实现展开，通过对比两种不同的实现方式，深入理解了 GPU 并行计算的核心原理和性能优化策略。

## 2. 实现方案分析

### 2.1 朴素矩阵乘法 (matmul_naive.cu)

**核心实现思路：**
- 每个线程负责计算结果矩阵 C 中的一个元素
- 线程通过全局内存直接访问矩阵 A 和 B 的元素
- 使用三重循环完成矩阵乘法的基本计算

**关键代码解析：**
```cuda
__global__ void matmul_naive(float *A, float *B, float *C, int M, int N, int K) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    if (row < M && col < N) {
        float sum = 0.0f;
        for (int k = 0; k < K; k++) {
            sum += A[row * K + k] * B[k * N + col];
        }
        C[row * N + col] = sum;
    }
}
```

**性能特点：**
- 实现简单直观，易于理解
- 但存在严重的全局内存访问瓶颈
- 每个线程需要多次访问全局内存，导致内存带宽成为性能瓶颈

### 2.2 共享内存优化矩阵乘法 (matmul_shared.cu)

**核心实现思路：**
- 引入共享内存 (shared memory) 作为中间缓存
- 将矩阵分块处理，每次处理一个块大小 (16x16) 的数据
- 利用共享内存的高带宽特性减少全局内存访问

**关键代码解析：**
```cuda
__global__ void matmul_shared(float *A, float *B, float *C, int M, int N, int K) {
    __shared__ float As[16][16];
    __shared__ float Bs[16][16];

    int bx = blockIdx.x;
    int by = blockIdx.y;
    int tx = threadIdx.x;
    int ty = threadIdx.y;

    int row = by * 16 + ty;
    int col = bx * 16 + tx;

    float sum = 0.0f;
    for (int k = 0; k < K; k += 16) {
        // 加载数据到共享内存
        if (row < M && (k + tx) < K) {
            As[ty][tx] = A[row * K + k + tx];
        } else {
            As[ty][tx] = 0.0f;
        }
        if (col < N && (k + ty) < K) {
            Bs[ty][tx] = B[k + ty * N + col];
        } else {
            Bs[ty][tx] = 0.0f;
        }
        __syncthreads();

        // 块内计算
        for (int i = 0; i < 16; i++) {
            sum += As[ty][i] * Bs[i][tx];
        }
        __syncthreads();
    }
    if (row < M && col < N) {
        C[row * N + col] = sum;
    }
}
```

**性能优化点：**
- 使用 `__shared__` 关键字声明共享内存变量
- 通过分块策略减少全局内存访问次数
- 利用 `__syncthreads()` 确保线程间同步
- 实现了数据复用，提高了内存访问效率

## 3. 技术原理深入理解

### 3.1 CUDA 线程模型

- **线程层次结构**：Grid → Block → Thread
- **线程索引计算**：通过 `blockIdx` 和 `threadIdx` 计算全局线程索引
- **二维线程布局**：使用 `dim3` 类型创建二维线程块和网格，适合矩阵运算

### 3.2 内存层次结构

| 内存类型 | 访问速度 | 作用范围 | 特点 |
|---------|---------|---------|------|
| 寄存器 | 最快 | 线程私有 | 自动分配，速度极快 |
| 共享内存 | 很快 | 线程块共享 | 用户管理，高带宽 |
| 全局内存 | 较慢 | 设备全局 | 容量大，访问延迟高 |

### 3.3 矩阵乘法的并行化策略

- **数据并行**：每个线程负责计算结果矩阵的一个元素
- **分块计算**：将大矩阵分解为小矩阵块，利用共享内存提高访问效率
- **内存访问模式优化**：确保内存访问的合并性，提高内存带宽利用率

## 4. 性能对比分析

| 实现方式 | 内存访问模式 | 全局内存访问次数 | 性能特点 |
|---------|-------------|-----------------|----------|
| 朴素实现 | 随机访问 | 每个元素多次访问 | 实现简单，性能较低 |
| 共享内存优化 | 合并访问 + 共享内存 | 大幅减少 | 实现复杂，性能显著提升 |
